Form input widths in ems to allow them to grow when text is zoomed

Test case 1 - comparison of input widths set in percentage and ems

Type test text in each and then increase font size to 200%.

Test case 2 - inputs with widths set in ems

Background

Widths for single line text inputs are set in percentages of the container in the standard GOV.UK toolkit. When text only is zoomed, the input does not grow, so any text entered may be hidden beyond the width of the input.

This suggested fix replaces the percentage widths with ems. Styles are then set with classes such as form-control-char-35.

Advantages

Assuming the community cannot find problems with this approach:

  • inputs scale with font size, so users browsing with larger font size don't have text clipped by the input box
  • class names follow likely character widths, so it's easier to set the expected length
  • it fits in with the aim to remove the 50% default width from existing styles

Suggested general test method

Objective: the page is readable and functional when the text size is doubled.

Method and tools

Important - use just text zoom and not page zoom (where everything gets proportionately larger).

Tool: easiest way is to install Zoom Page add-in for Firefox:
https://addons.mozilla.org/en-us/firefox/addon/zoom-page/

View page and choose 200%.

The test is - is the page still readable and functional? Typical problems are when boxes have a fixed height and text does not fit.

The test fails if:

  • text inside input boxes is partly hidden
  • there is a horizontal scroll bar
  • text inside divs or HTML5 sections is clipped

Guideline: 1.4.4 Resize text (Level AA)
